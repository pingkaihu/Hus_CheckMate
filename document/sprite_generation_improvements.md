# Sprite Generation Pipeline Improvements

我們已經成功解決了由 AI 生成靜態圖片轉換為動畫精靈圖 (Sprite Sheet) 過程中的幾項核心品質問題。以下是本次更新的總結。

## 1. 解決「人物大小忽大忽小與上下抖動」問題
**問題原因**：
原本的處理腳本在去背後，會使用 `getbbox()` 將圖片緊密裁切，並將底部對齊。但由於 AI 生成的每格人物動作不同（例如抬腳），這會導致裁切框的底部高度改變，強制對齊後反而造成人物重心忽上忽下。

**解決方案**：
在 `from-image-to-sprite.py` 腳本中新增了 `--no-crop` 參數。
* 開啟此參數後，程式會完全信任 AI 產生的原始網格排版，略過 Bounding Box 裁切。
* 這保留了人物在畫布中的絕對位置，徹底消除了上下抖動與大小劇烈變動的錯覺。

## 2. 解決「邊緣紫邊與狗牙鋸齒」問題
**問題原因**：
AI 在生成 `#FF00FF` (Magenta) 背景時，人物與背景交界處會產生平滑的抗鋸齒 (Anti-aliasing) 漸層。原本的演算法是「二分法硬裁切」，無法處理這種漸層，導致留下充滿鋸齒的紫紅色光暈。

**解決方案**：
大幅強化了 `from-image-to-sprite.py` 的 `remove_bg_color` 去背演算法：
1. **Alpha Matting (漸變透明)**：偵測到邊緣漸層時，依據顏色距離給予半透明效果 (Soft Edge)，讓邊緣變得平滑。
2. **Defringe / Spill Suppression (終極消光)**：加入電影綠幕去背等級的溢色抑制演算法。程式會掃描所有邊緣像素，只要偵測到帶有紫紅色（紅色與藍色數值高於綠色），就會強制將紅藍通道向綠色（真實亮度）靠攏 (`R = min(R, G)`)，從數學上 100% 抹除了紫紅色殘留。

## 3. 最佳實踐建議 (Best Practices)
未來在使用 `from-image-to-sprite` 生成精靈圖時，建議遵循以下參數設定以獲得最佳品質：
* **減少幀數**：建議使用 **4 或 6 幀**，降低 AI 生成大網格時的比例失真機率。
* **使用新參數**：
  ```bash
  --frames 6 --cols 3 \
  --bg-mode magenta --bg-threshold 80 \
  --align bottom --shared-scale \
  --no-crop
  ```
